# Interview

## 常用的反问问题
- **当前业务中模型的迭代周期和上线压力**
    - 这个问题主要是想了解公司对于当前业务的重视程度，以及对于模型迭代的需求和频率。迭代周期短、上线压力大的业务通常意味着需要快速响应市场变化和用户需求，可能需要更高效的模型训练和部署流程。
- **团队规模和基建**
    - 这个问题主要是想了解一个业务每个人需要负责的工作量，以及能够调用的资源。团队规模较大、基建完善的业务通常意味着有更多的人力和技术资源支持模型的开发和部署，可能会有更好的协作和分工。基建包含数据基建和显卡资源。如果涉及组织架构不能明说，可以举例问一下，比如做一次模型迭代单人所能调用的显卡数量。
- **业务内容和技术栈**
    - 这个问题主要是想了解实习省可以负责的业务的具体内容和使用的技术栈。业务内容可以帮助面试者了解模型应用的场景和需求，技术栈可以帮助面试者了解团队使用的工具和框架，以及是否与自己的经验匹配。
